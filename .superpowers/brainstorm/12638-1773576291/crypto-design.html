Секция 5: E2E шифрование

Криптографические примитивы и ключевой обмен

Схема ключей

// 1. Из BIP39 seed phrase → мастер-ключ
seed_phrase (24 words)
    │
    ▼  HKDF-SHA256
┌──────────────────────┐
│  master_secret       │
│  (32 bytes)          │
└──────┬───────────────┘
       │
       ├──▶ Ed25519 keypair        // identity (подписи, NodeId)
       │     ├── public_key         // = твой идентификатор
       │     └── secret_key         // хранится локально
       │
       └──▶ X25519 keypair         // encryption (DH key exchange)
             ├── public_key         // отправляется в Hello
             └── secret_key         // хранится локально

// Ed25519 → X25519 конверсия стандартная (RFC 7748)

Шифрование сообщения

// Для каждой пары (Alice, Bob) вычисляется shared secret:

shared_secret = X25519_DH(alice_x25519_secret, bob_x25519_public)
    │
    ▼  HKDF-SHA256 (salt = sorted_pubkeys)
    │
symmetric_key (32 bytes)

// Шифрование каждого сообщения:

nonce = random_24_bytes()  // уникальный для каждого сообщения

ciphertext = XChaCha20-Poly1305(
    key   = symmetric_key,
    nonce = nonce,
    aad   = message_id,    // associated data — защита от подмены
    data  = plaintext
)

// Отправляем: { id, ciphertext, nonce, timestamp }
// symmetric_key НИКОГДА не передаётся по сети

Rust crates

# Cargo.toml (dchat-core)

bip39          = "2"        # seed phrase генерация/парсинг
ed25519-dalek  = "2"        # Ed25519 подписи
x25519-dalek   = "2"        # X25519 DH key exchange
chacha20poly1305 = "0.10"   # XChaCha20-Poly1305 AEAD
hkdf           = "0.12"     # HKDF key derivation
sha2           = "0.10"     # SHA-256 для HKDF
rand           = "0.8"      # nonce генерация

Свойства безопасности

  • Confidentiality — XChaCha20-Poly1305 AEAD, только участники могут читать
  • Integrity — Poly1305 MAC + message_id как AAD, подмена детектируется
  • Authentication — Ed25519 identity привязан к iroh NodeId, подделка невозможна
  • Нет forward secrecy — для MVP ок. Double Ratchet добавим позже
  • Replay protection — UUID сообщения + проверка дубликатов в SQLite
