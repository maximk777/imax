Секция 4: Wire Protocol

Протокол обмена сообщениями поверх iroh

Формат пакета

// Все сообщения сериализуются через serde + postcard (компактный бинарный формат)

enum WireMessage {
    // --- Handshake (первое подключение) ---
    Hello {
        public_key: [u8; 32],       // Ed25519 pubkey
        nickname: String,           // отображаемое имя
        protocol_version: u8,       // версия протокола (1)
    },

    // --- Сообщения ---
    ChatMessage {
        id: Uuid,                   // уникальный ID
        ciphertext: Vec<u8>,        // зашифрованный контент
        nonce: [u8; 24],            // nonce для XChaCha20
        timestamp: u64,             // unix millis
    },

    // --- Подтверждения ---
    Ack {
        message_id: Uuid,           // какое сообщение подтверждаем
        status: AckStatus,          // delivered | read
    },

    // --- Синхронизация оффлайн ---
    SyncRequest {
        last_seen_id: Option<Uuid>, // последнее известное сообщение
    },
    SyncResponse {
        messages: Vec<ChatMessage>, // пропущенные сообщения
    },

    // --- Присутствие ---
    Ping,
    Pong,
}

enum AckStatus {
    Delivered,
    Read,
}

Транспорт

// iroh использует ALPN для мультиплексирования протоколов
// Регистрируем свой протокол:

const ALPN: &[u8] = b"dchat/1";

// Каждое соединение — bidirectional QUIC stream
// Формат кадра:

┌──────────────┬───────────────────┐
│ length: u32  │ payload: [u8]     │
│  (4 bytes)   │  (postcard bytes) │
└──────────────┴───────────────────┘

// length-prefixed framing — простой и надёжный

Flow подключения

 Alice                              Bob
  │                                   │
  │──── QUIC connect (ALPN=dchat/1) ─▶│
  │                                   │
  │──── Hello { pubkey, nick, v=1 } ─▶│
  │◀─── Hello { pubkey, nick, v=1 } ──│
  │                                   │
  │  // обе стороны теперь знают pubkey друг друга
  │  // X25519 shared secret = DH(alice_priv, bob_pub)
  │                                   │
  │──── SyncRequest { last_seen } ───▶│
  │◀─── SyncResponse { missed } ──────│
  │                                   │
  │◀═══ ChatMessage (encrypted) ═════▶│
  │◀═══ Ack ═════════════════════════▶│
  │                                   │
  │──── Ping ────────────────────────▶│  // каждые 30 сек
  │◀─── Pong ─────────────────────────│

Ключевые решения

  • postcard — компактный бинарный формат, идеально для Rust (serde, no_std совместимый)
  • Length-prefixed framing — 4 байта длины + payload. Просто парсить, нет проблем с разделителями
  • XChaCha20-Poly1305 — шифрование сообщений. Быстрый, безопасный, 24-byte nonce (нет проблем с повторами)
  • Ping/Pong — keepalive каждые 30 сек для обнаружения обрыва
  • SyncRequest/Response — при reconnect досылка пропущенных сообщений
  • protocol_version — в Hello, для обратной совместимости в будущем
